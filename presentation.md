LURNSTACK ADMIN WORKFLOW DOCUMENTATION

This document provides a clean and professional overview of the completed modules and workflows in the LurnStack Admin panel. It outlines only the active systems that are implemented and ready for use.


ADMINISTRATIVE IDENTITY GATEWAY

The login and registration system secures administrative control over the platform.

Admin Registration
New administrators can create accounts by providing their full name, email address, and a secure password.

Admin Authentication
Registered administrators sign in using their credentials. The system verifies their login and stores a secure security token in the local browser database.

Automatic Protection
All administrative pages are private. The system detects if an admin session is active. If not, it blocks access and redirects the visitor back to the login gateway.


OPERATIONAL DASHBOARD

The dashboard serves as the administrative landing page, offering quick metrics about platform health.

Live Metric Counts
Admins can view the exact total count of registered students and trainers active on the platform.

Status Tracking
The dashboard alerts the user to when the numbers were last updated from the live backend server.


STUDENT DATABASE MANAGEMENT

Admins can review and clean the registry of students enrolled in courses.

Detailed Roster
The student directory displays a list showing each student's full name, email address, contact phone number, and their exact registration date.

Roster Updates
Admins have the authority to remove student records from the database using confirmation alerts.


INSTRUCTOR MANAGEMENT

Admins can view and manage teaching permissions for trainers.

Trainer Directory
Lists all trainers registered to host courses and classes.

Instant Access Control
Admins can activate or deactivate trainers instantly using inline toggle switches. Deactivating a trainer prevents them from scheduling new classes.

Account De-registration
Admins can remove trainers from the platform entirely when required.


LIVE CLASSROOM COORDINATION

The scheduling center manages live interactive learning sessions.

Dynamic Roster
Shows all scheduled classes, assigned instructors, and dates. Admins can search by title or filter classes by course name and current status.

Lifecycle Status
Sessions are classified into three phases: Scheduled for future classes, Live for sessions currently in progress, and Completed for past classes.

Event Scheduling
Admins can schedule new classes or edit existing ones, entering details like course name, topic, instructor, date, start time, duration, Google Meet link, and thumbnail images.

Class Deletion
Allows administrators to remove scheduled live classes from the system.
